Implementa el código y las pruebas de aceptación de una especificación.
Si el usuario no proporciona una especificación, toma la primera especificación no implementada del BACKLOG.
Usa la skill `planning-spec` para generar los planes de implementación.
Sigue cada plan paso a paso, ejecutando y marcando sus tareas como completadas.
Al terminar, usa la skill `releasing-spec` para liberar la especificación.
