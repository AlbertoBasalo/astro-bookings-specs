# Writing prompt

Follow this instructions to write a well formatted and clear command prompt.

## When to Use

Use it for creating, updating or refining command prompts.

## Template 

Use this template as guide for the outline and format of the file.

```md
# {Command prompt name}

## Role
{short description in one sentence}

## Context
{short paragraph explaining the expected output}

### References
- {list of files or folders involved}
### Skills
- {list of skills potentially useful}
### Tools
- {list of tools potentially useful}

## Task
{short paragraph explaining the goal of the task to do}

## Constraints
{short paragraph about optional limits or restrictions}

## Steps

### 1. {Step 1 title}
{short description in one sentence}
- [ ]{task 1.1}
- [ ]{task 1.n}

### n. {Step n title}
{short description in one sentence}
- [ ]{task n.1}
- [ ]{task n.n}

## Output

{short paragraph explaining the expected output}

## Verification

- [ ]{Criteria 1}
- [ ]{Criteria n}

```

## Output
Before writing the prompt, be sure to:
- Write a terse, concise and clear command prompt in markdown format.
- Reduce verbose phrasing while preserving behavior and downstream intent.
- Ensure the prompt is easy to customize and adapt to different needs and contexts.
